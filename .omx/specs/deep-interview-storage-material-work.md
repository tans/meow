## 意图 (Why)
构建一个基于 Rustfs 的统一存储系统，支持小程序和 Web 端的素材与作品上传、预览和下载功能。
保护创作者权益：商家可在审核前预览作品内容，但只有在审核通过后才能下载完整质量的正式版作品。

---

## 期望结果 (Desired Outcome)
1. 用户可上传素材和作品
2. 用户可查看素材和作品的预览版
3. 作品预览版与正式版分离：
   - 预览版：低质量，在线播放/查看，前端不提供下载按钮
   - 正式版：完整质量，审核通过后可下载
4. 正式版作品只有在审核通过后才能被商家和创作者下载

---

## 范围 (In-Scope)

### 功能范围
1. **素材管理**（商家上传的任务附件）
   - 支持图片、视频、文档上传
   - 开放访问，可直接下载

2. **作品管理**（创作者提交的投稿内容）
   - 上传时存储完整原文件
   - 系统自动生成预览版：
     - 图片：压缩至 480px 宽度
     - 视频：转码至原始码率的 30%
   - 通过预签名 URL 提供访问

3. **访问控制**
   - 预览版：开放查看权限，在线播放
   - 正式版：审核通过后，商家和创作者可下载

4. **存储层**
   - 新部署 Rustfs 作为底层存储
   - S3 兼容 API 访问
   - 预签名 URL 机制

### 技术范围
- API 层：Rustfs 集成、预签名 URL 生成、访问权限控制
- 预览版生成：图片压缩、视频转码
- 数据库：存储文件元数据（URL、类型、状态）
- 前端：预览组件、下载按钮控制

## 可测试验收标准 (Testable Acceptance Criteria)

### 功能验收
- [ ] 商家可以上传素材文件（图片/视频/文档）
- [ ] 上传的素材可以通过预签名 URL 访问和下载
- [ ] 创作者可以上传作品文件
- [ ] 上传后自动生成预览版：

  - [ ] 视频：码率为原始的 30%，保持格式
- [ ] 预览版可通过预签名 URL 在线播放/查看
- [ ] 作品审核通过前，正式版不可下载（返回 403）
- [ ] 作品审核通过后，商家可通过预签名 URL 下载正式版
- [ ] 作品审核通过后，创作者可通过预签名 URL 下载正式版
- [ ] 前端预览组件仅显示预览版，不显示正式版下载按钮（审核前）

- [ ] 预签名 URL 包含正确的 Content-Type

- [ ] 不支持的文件类型返回 400 Bad Request

### 性能验收
- [ ] 预览版生成在 30 秒内完成（500MB 视频）
- [ ] 预签名 URL 生成响应时间 < 100ms
