# Context Snapshot: www 项目官网

## Task Statement
创建一个 www 项目，展示项目官网，并且有商家后台(admin)的入口和 web 入口。

## Desired Outcome
一个独立的官网应用，用于：
1. 展示项目/产品介绍
2. 提供到 admin（商家后台）的入口
3. 提供到 web（创作者/需求方工作台）的入口

## Stated Solution
创建 apps/www 应用

## Probable Intent Hypothesis
用户需要一个营销/展示性质的官网，作为用户访问的第一站，引导不同类型的用户进入不同的应用：
- 商家/运营人员 → admin
- 创作者/需求方 → web

## Known Facts/Evidence
- 现有 monorepo 结构：apps/web, apps/admin, apps/api 等
- web: 创作者/需求方工作台（已迁移到 shadcn/ui）
- admin: 商家后台（已迁移到 shadcn/ui）
- 技术栈：React + Vite + TypeScript + shadcn/ui + Tailwind

## Constraints
- 使用现有技术栈（React + Vite + shadcn/ui）
- 保持与现有项目一致的风格
- 部署后独立访问

## Unknowns/Open Questions
1. 官网的具体内容（ landing page 结构）
2. 设计风格（是否沿用蓝色主题）
3. 入口的具体形式（按钮、卡片、导航？）
4. 是否需要响应式设计
5. 是否需要其他页面（关于我们、功能介绍、价格等）
6. 域名/路径规划

## Decision-Boundary Unknowns
1. 我可以决定哪些设计细节？
2. 哪些内容需要用户确认？

## Likely Codebase Touchpoints
- apps/www/ (new)
- pnpm-workspace.yaml
- turbo.json
