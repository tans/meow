# 交付路线

## Phase 0: Foundation

- 完成 monorepo、workspace、TypeScript、Turbo 基础设施。
- 固化三端壳、bounded context 和流程清单。
- 让 `harness` 能覆盖文档里的三条关键闭环。

## Phase 1: MVP

- 用户端先落商家任务发布、任务列表、投稿审核、创作者任务浏览和投稿。
- 后台先落任务管理、用户管理、申诉处理和资金总览。
- 后端最小可用能力包括：
  - 账号与会话
  - 任务 CRUD
  - 投稿 CRUD
  - 托管账户与结算
  - 基础申诉流转

## Phase 2: Beta

- 接入 AI 查重、违规检测和异常提醒。
- 补全评分、公开评价、额外打赏和爆款标记。
- 上线每日/月度对账、提现审核和更细粒度权限管理。

## Phase 3: GA

- 做完整数据看板、版权归档、系统公告和运营工具。
- 建立全链路审计、告警、回滚与数据修复工具。
- 将 `harness` 从静态契约校验扩展为接口回放和端到端验收。

## 优先级建议

先做这几项：

1. 任务发布 + 托管预算。
2. 投稿审核 + 基础奖励发放。
3. 任务结束自动结算 + 未用完赏金退回。
4. 创作者收益解冻 + 提现。
5. 后台申诉裁决 + 信用回写。

不要一开始就做这几项：

1. 复杂数据大屏。
2. 过度细化的营销能力。
3. 完整版权协商工作流。
4. 过重的低代码配置系统。
